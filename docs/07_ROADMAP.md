# Дорожная карта реализации — Радио «Толкай Вода»

10 этапов от пустого репозитория до прод-готовности. Каждый этап заканчивается
**демонстрируемым результатом**.

Ключевая идея: **MVP-first**. Этап 1 = живой радио-плеер на главной. Всё остальное
наслаивается без перепиливания базы.

> **Где смотреть прогресс**: текущий статус всех этапов — в [`STATUS.md`](STATUS.md).
> Внутри каждого этапа ниже — чекбоксы acceptance-пунктов. Закрытие этапа
> сопровождается git-тегом `stage-N-done`.

## Архитектурное решение (важный пункт ТЗ — нагрузка 100 юзеров на 2 ядра / 2 ГБ)

Плеер играет **случайный трек у каждого юзера** (не синхронный эфир), поэтому
HLS/FFmpeg-стрим **не нужен**. mp3 лежат на **S3 Beget** и раздаются через **CDN**
(`ceshocofureled.begetcdn.cloud`). Сайт играет mp3 **напрямую с CDN** — сервер не
трогает аудио-байты вообще. Всю аудио-полосу держит CDN, сервер (2 ядра / 2 ГБ)
обслуживает только лёгкий API блога/прогноза и Telegram-бота.

Стек: **Node.js + Fastify + better-sqlite3**, фронт — HTML/CSS/Vanilla JS,
reverse-proxy **Caddy** (авто-HTTPS), **docker-compose**, деплой **GitHub Actions**.

## Этап 1 — Радио-плеер (главная вертикаль)

**Цель**: живой плеер на главной по дизайн-макету.

**Что делаем**:
- Вёрстка главной: фон (озеро), лого по центру, заголовок «РАДИО ТОЛКАЙ ВОДА»
- Шрифты Oswald (заголовки) + Inter (текст), self-hosted woff2
- Кнопка «СЛУШАТЬ РАДИО» → случайный трек, по `ended` → следующий случайный
- Анимация звуковой дорожки (WebAudio AnalyserNode или CSS-fallback)
- Карточка «Сейчас играет»: обложка, название, исполнитель
- API `GET /api/tracks` (пока из локального манифеста-заглушки)

**Критерии приёмки** (закрытие → tag `stage-1-done`):
- [x] Главная визуально соответствует макету (фон, лого, заголовок, кнопка)
- [x] Клик по кнопке запускает случайный трек, звук играет
- [x] При воспроизведении видна анимация дорожки, на паузе — замирает
- [x] По окончании трека автоматически стартует следующий случайный
- [x] «Сейчас играет» показывает название текущего трека

## Этап 2 — S3/CDN: загрузка треков

**Цель**: плеер играет реальные песни клиента с CDN, не с сервера.

**Что делаем**:
- Скрипт распаковки `Радио Толкай Вода.zip`, нормализация имён (slug/uuid)
- Загрузка mp3 на S3 Beget (бакет `incomparable-darian`)
- Генерация `tracks.json` (название из тега/имени файла + CDN-URL)
- `GET /api/tracks` отдаёт реальный манифест

**Критерии приёмки** (закрытие → tag `stage-2-done`):
- [x] 50 mp3 доступны по CDN-URL (HTTP 200, audio/mpeg)
- [x] `tracks.json` содержит все треки с корректными названиями
- [x] Плеер на главной играет реальные треки с CDN
- [x] Аудио грузится с CDN-домена (проверка в Network), не с сервера

## Этап 5 — Telegram-бот контента

**Цель**: контент попадает на сайт из Telegram.

**Что делаем**:
- Бот (grammY/Telegraf, long-polling), whitelist разрешённых ID
- Личка: фото+текст. Подпись с «Прогноз» → обновляет прогноз дня; иначе → пост блога
- Загрузка фото из Telegram на S3, сохранение записи в SQLite
- Логирование pino (на русском: дата, username, id, действие)

**Критерии приёмки** (закрытие → tag `stage-5-done`):
