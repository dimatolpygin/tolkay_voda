## Активная работа

**Этап**: 5 «Telegram-бот» — переделан под FSM + ИИ, ждёт BOT_TOKEN/BOT_ADMIN_IDS/OPENAI_API_KEY для живой проверки.
**Ветка**: `dev`
**Что сделано (итерация 2, по правкам клиента)**:
- FSM с инлайн-кнопками вместо ключевых слов: /start → [Прогноз дня][Статья] → текст → фото/без фото → предпросмотр → [Опубликовать][Отмена] (`src/bot/index.js`, состояние по userId).
- ИИ-выжимка прогноза через OpenAI gpt-4o-mini (`src/lib/ai.js`): сырой текст → {subtitle, intro, water, color, food, advice, full}. Есть фолбэк без ключа.
- Схема forecast расширена колонками subtitle/intro/water/color/food/advice (идемпотентная миграция в `db.js`).
- Фронт прогноза переделан по макету: лунный день, вводка, строки Вода/Цвет/Еда/Совет дня (золотые ключи), модалка с секциями (## заголовки). Проверено в браузере (Playwright) — блок и модалка по макету, консоль чистая.
- Фото Telegram → S3 → CDN (`media.js`), CDN-URL показывается в ответе бота после публикации.
- whitelist, pino-логи на русском (вход/отказ/ответ/загрузка фото). Старый `parse.js` удалён.
**Осталось для закрытия**: заполнить `.env` (BOT_TOKEN, BOT_ADMIN_IDS, OPENAI_API_KEY) и вживую прогнать: FSM-сценарий, ИИ-выжимка реального прогноза, фото на CDN, игнор чужого id, логи.

## Протокол обновления

**При старте сессии**:
1. Прочитать этот файл.
2. `git tag -l "stage-*"` и сверить с колонкой «Закрывающий тег».
3. `git log --oneline -10` — что менялось последним.
4. Только после этого приступать к работе.

**При взятии этапа в работу**: обновить «Текущий этап»/«Активная работа», статус 🚧, коммит `chore(status): start stage N`.

**При закрытии этапа**: все чекбоксы `[x]` и проверены → `git tag stage-N-done` → обновить таблицу и «Историю закрытий» → коммит `chore(status): close stage N`.

**Если этап заблокирован**: статус ⏸, описать блокер и что нужно для разблокировки.
