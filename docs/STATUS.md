## Активная работа

**Этап**: 5 «Telegram-бот» — код готов, ждёт BOT_TOKEN/BOT_ADMIN_IDS для живой проверки.
**Ветка**: `dev`
**Что сделано**: grammY-бот (`src/bot/`), whitelist по BOT_ADMIN_IDS, pino-логи на русском. Протокол: сообщение с «Прогноз» в первой строке → upsert прогноза на сегодня; иначе первая строка = заголовок → новый пост блога. Фото из Telegram → S3 → CDN (`src/bot/media.js`). Модули: `parse.js` (разбор), `slug.js` (транслит/уникальный slug), `store.js` (запись в БД). Оффлайн-тесты парсера/slug/store — 13 проверок зелёные. Бот без токена корректно завершается с ошибкой.
**Осталось для закрытия**: заполнить `.env` и вживую прогнать критерии приёмки (прогноз, пост, фото на CDN, игнор чужого id, логи).

## Протокол обновления

**При старте сессии**:
1. Прочитать этот файл.
2. `git tag -l "stage-*"` и сверить с колонкой «Закрывающий тег».
3. `git log --oneline -10` — что менялось последним.
4. Только после этого приступать к работе.

**При взятии этапа в работу**: обновить «Текущий этап»/«Активная работа», статус 🚧, коммит `chore(status): start stage N`.

**При закрытии этапа**: все чекбоксы `[x]` и проверены → `git tag stage-N-done` → обновить таблицу и «Историю закрытий» → коммит `chore(status): close stage N`.

**Если этап заблокирован**: статус ⏸, описать блокер и что нужно для разблокировки.
